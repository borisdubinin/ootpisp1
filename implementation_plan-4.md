# Implementation Plan: Графический редактор фигур (C++ / SFML)

---

## Общее видение

Приложение — это минималистичный **векторный графический редактор** в духе Figma/Inkscape, но упрощённый. Пользователь видит бесконечный (или большой) холст, на котором расположены фигуры. Интерфейс разделён на зоны: панель инструментов, холст, панель свойств. Всё управление через мышь + клавиатуру.

### Фаза 1 — Архитектура классов (2–3 дня)

**Цель:** чистая иерархия, не зависящая от рендеринга.

#### 1.1 Базовый класс `Figure`

```cpp
class Figure {
public:
    // Точка привязки (абсолютные координаты на холсте)
    sf::Vector2f anchor;

    // Цвет заливки
    sf::Color fillColor;

    // Виртуальные границы (bounding box)
    sf::FloatRect getBoundingBox() const;

    // Вершины (относительно anchor)
    virtual std::vector<sf::Vector2f> getVertices() const = 0;

    // Отрисовка
    virtual void draw(sf::RenderTarget& target) = 0;

    // Проверка попадания курсора
    virtual bool contains(sf::Vector2f point) const = 0;

    // Перемещение
    void move(sf::Vector2f delta);

protected:
    struct Edge {
        float width;       // ширина линии
        sf::Color color;   // цвет линии
    };
    std::vector<Edge> edges;
};
```

#### 1.2 Дочерние классы

| Класс | Особенности |
|---|---|
| `Rectangle` | 4 вершины, 4 грани |
| `Triangle` | 3 вершины, 3 грани |
| `Hexagon` | 6 вершин, 6 граней |
| `Rhombus` | 4 вершины, задаются диагонали |
| `Trapezoid` | 4 вершины, две параллельные стороны |

Каждый класс реализует `getVertices()`, `draw()`, `contains()`.

#### 1.3 Класс `Scene`

```cpp
class Scene {
    std::vector<std::unique_ptr<Figure>> figures;
    Figure* selectedFigure = nullptr;
public:
    void addFigure(std::unique_ptr<Figure> fig);
    void removeFigure(Figure* fig);
    Figure* hitTest(sf::Vector2f point);
    void drawAll(sf::RenderTarget& target);
};
```

### Фаза 2 — Рендеринг фигур (3–4 дня)

**Цель:** корректная отрисовка фигур с заливкой и цветными гранями.

#### 2.1 Заливка

Использовать `sf::ConvexShape` для заливки внутренней области.

#### 2.2 Рендеринг граней с учётом ширины

SFML рисует линии через `sf::RectangleShape` (развёрнутый вдоль ребра прямоугольник):

```
Для каждого ребра (A → B):
  1. Вычислить вектор направления и нормаль
  2. Построить прямоугольник шириной edge.width
  3. Залить цветом edge.color
```

#### 2.3 Корректная стыковка линий (ключевой нюанс)

Когда две грани встречаются в вершине и имеют **разную ширину / цвет**, нужно:

1. Вычислить биссектрису угла между двумя рёбрами
2. Обрезать оба прямоугольника по биссектрисе (miter join)
3. Заполнить треугольный «зазор» одним из цветов (по договорённости — цветом более широкой линии)

Алгоритм:
```
Для каждой вершины Vi:
  ребро1 = (V(i-1) → Vi),  ребро2 = (Vi → V(i+1))
  найти точку пересечения внешних границ двух рёбер
  → это точка miter
  если miter слишком далеко (острый угол) — обрезать (bevel)
```

#### 2.4 Виртуальные границы (bounding box)

Отрисовывать пунктирным прямоугольником вокруг выделенной фигуры:

```cpp
sf::RectangleShape bbox;
bbox.setFillColor(sf::Color::Transparent);
bbox.setOutlineColor(sf::Color(0, 120, 215)); // синий как в Windows
bbox.setOutlineThickness(1.f);
// + уголки-маркеры для визуализации
```

### Фаза 4 — Управление и интерактивность (2–3 дня)

**Цель:** мышь + клавиатура работают плавно и предсказуемо.

#### 4.1 Выделение фигуры

```
MouseButtonPressed (Left):
  fig = scene.hitTest(mousePos)
  if fig != null:
    selectedFigure = fig
    dragOffset = mousePos - fig.anchor
    state = DRAGGING
  else:
    selectedFigure = null
```

#### 4.2 Перетаскивание

```
MouseMoved (while DRAGGING):
  selectedFigure.anchor = mousePos - dragOffset
  // проверить bounding box — не выходить за границы окна

MouseButtonReleased:
  state = IDLE
```

#### 4.3 Создание новой фигуры

```
Активен инструмент (не Selection):
MouseButtonPressed → запомнить startPos
MouseMoved        → показать preview фигуры (полупрозрачно)
MouseButtonReleased → создать фигуру с anchor = центр выделенной области
```

### Фаза 7 — Точка привязки (anchor point) (1–2 дня)

**Цель:** каждая фигура имеет визуально редактируемую точку привязки, как pivot point в After Effects.

#### 7.1 Что такое точка привязки в контексте проекта

- Это абсолютная точка на холсте, к которой "прикреплена" фигура
- Все координаты вершин хранятся **относительно неё** (могут быть отрицательными)
- При перемещении фигуры мышью — двигается точка привязки, вершины следуют за ней
- При изменении координат точки привязки в панели свойств — фигура перемещается на холсте

#### 7.2 Визуальное отображение

Точка привязки отображается на холсте как специальный маркер (только при выделении фигуры):

```
  ───┼───   кружок с перекрестием
     │       (как в After Effects / Figma)
```

Реализация:
```cpp
void drawAnchorMarker(sf::RenderTarget& target, sf::Vector2f pos) {
    // Кружок
    sf::CircleShape circle(6.f);
    circle.setFillColor(sf::Color::Transparent);
    circle.setOutlineColor(sf::Color::White);
    circle.setOutlineThickness(1.5f);
    circle.setOrigin(6.f, 6.f);
    circle.setPosition(pos);
    // Перекрестие (две линии)
    // ...
    target.draw(circle);
}
```

#### 7.3 Перетаскивание точки привязки

Точка привязки перетаскивается **отдельно от фигуры** — например, при зажатом `Alt` + drag по маркеру:

```
Alt + MouseButtonPressed на маркере якоря:
  state = DRAGGING_ANCHOR

MouseMoved (while DRAGGING_ANCHOR):
  delta = mousePos - oldAnchorPos
  figure.anchor += delta
  // Пересчитать относительные координаты всех вершин:
  for each vertex:
    vertex -= delta   // вершины остаются на месте визуально,
                      // но их относительные координаты меняются
```

Ключевой момент: при перетаскивании якоря **фигура визуально не двигается** — двигается только точка привязки, а относительные координаты вершин пересчитываются автоматически.

#### 7.4 В панели свойств

```
┌─────────────────────────────┐
│  ТОЧКА ПРИВЯЗКИ             │
│  X: [_______]  Y: [_______] │  ← абсолютные координаты
│  (изменение = фигура двигается на холсте)
└─────────────────────────────┘
```

### Фаза 8 — Редактирование вершин на холсте (Node Editor) (2–3 дня)

**Цель:** каждую вершину фигуры можно перетащить мышью прямо на холсте, свободно меняя форму фигуры.

#### 8.1 Концепция

Это аналог **Node Tool** в Inkscape или редактора вершин в любом векторном редакторе. Квадрат можно превратить в трапецию, трапецию — в произвольный четырёхугольник, просто таща углы мышью.

#### 8.2 Режим редактирования вершин

Отдельный инструмент или режим, активируется например двойным кликом по фигуре или клавишей `N`:

```
Двойной клик по фигуре  →  входим в режим Node Edit
  - на каждой вершине появляется маркер (квадратик ~8px)
  - фигура подсвечивается иначе (например, пунктирный контур)

Клик + drag по маркеру вершины:
  vertex.relativePos = mousePos - figure.anchor
  фигура перестраивается в реальном времени

Escape / одинарный клик вне фигуры  →  выходим из Node Edit
```

#### 8.3 Визуальные маркеры вершин

```cpp
void drawVertexHandles(sf::RenderTarget& target, Figure* fig) {
    for (auto& v : fig->getVertices()) {
        sf::Vector2f absPos = fig->anchor + v; // абс. координата вершины
        sf::RectangleShape handle(sf::Vector2f(8.f, 8.f));
        handle.setOrigin(4.f, 4.f);
        handle.setPosition(absPos);
        handle.setFillColor(sf::Color::White);
        handle.setOutlineColor(sf::Color(0, 120, 215));
        handle.setOutlineThickness(1.5f);
        target.draw(handle);
    }
}
```

#### 8.4 Связь с панелью свойств

Когда вершина перетаскивается мышью — поля `dx`/`dy` в панели свойств обновляются в реальном времени. И наоборот: при ручном вводе в поле — вершина перемещается на холсте.

#### 8.5 Обновление bounding box

После каждого изменения вершины — bounding box пересчитывается автоматически:

```cpp
sf::FloatRect Figure::getBoundingBox() const {
    float minX = INF, minY = INF, maxX = -INF, maxY = -INF;
    for (auto& v : getVertices()) {
        sf::Vector2f abs = anchor + v;
        minX = std::min(minX, abs.x);
        minY = std::min(minY, abs.y);
        maxX = std::max(maxX, abs.x);
        maxY = std::max(maxY, abs.y);
    }
    return { minX, minY, maxX - minX, maxY - minY };
}
```

#### 8.6 Горячие клавиши Node Editor

| Клавиша | Действие |
|---|---|
| `N` | Войти в режим редактирования вершин |
| Двойной клик по фигуре | Войти в режим редактирования вершин |
| `Escape` | Выйти из режима |

### Фаза 9 — Вращение фигур (1–2 дня)

**Цель:** каждую фигуру можно вращать как мышью на холсте, так и вводом угла в панели свойств.

#### 9.1 Хранение угла в классе Figure

```cpp
class Figure {
    float rotationAngle = 0.f; // в градусах, 0–360
    // ...
};
```

Вращение применяется при вычислении абсолютных координат вершин:
```cpp
sf::Vector2f Figure::getAbsoluteVertex(sf::Vector2f relative) const {
    float rad = rotationAngle * M_PI / 180.f;
    float rx = relative.x * cos(rad) - relative.y * sin(rad);
    float ry = relative.x * sin(rad) + relative.y * cos(rad);
    return anchor + sf::Vector2f(rx, ry);
}
```

Ось вращения — **точка привязки**. Именно поэтому якорь важен: перенеси его в угол фигуры — и фигура будет вращаться вокруг угла, поставь в центр — вокруг центра.

#### 9.2 Вращение мышью на холсте

При выделении фигуры появляется **маркер вращения** — кружок чуть выше bounding box (как в Figma):

```
        ○          ← маркер вращения (над верхней стороной bbox)
  ┌─────────────┐
  │   фигура    │
  └─────────────┘
```

Логика:
```
MouseButtonPressed на маркере вращения:
  state = ROTATING
  startAngle = atan2(mousePos - figure.anchor)

MouseMoved (while ROTATING):
  currentAngle = atan2(mousePos - figure.anchor)
  figure.rotationAngle += currentAngle - startAngle
  startAngle = currentAngle

Зажат Shift при вращении → snap к углам кратным 15°
```

#### 9.3 Ввод угла в панели свойств

```
┌─────────────────────────────┐
│  ВРАЩЕНИЕ                   │
│  Угол: [_____] °            │
│  [↺ Сбросить]               │
└─────────────────────────────┘
```

Ввод числа → фигура поворачивается на холсте в реальном времени.

#### 9.4 Горячие клавиши

| Клавиша | Действие |
|---|---|
| `R` (зажать) + drag мышью | Быстрое вращение |
| `Shift` при вращении | Snap по 15° |

### Фаза 10 — Глобальная точка отсчёта (World Origin) (1–2 дня)

**Цель:** как в AutoCAD — пользователь может задать точку (0, 0) мирового пространства и видеть все координаты относительно неё. Плюс возможность сдвинуть весь холст так, чтобы (0, 0) оказался в нужном месте экрана.

#### 10.1 Концепция

Есть два уровня координат:
- **Screen space** — пиксели экрана, меняются при зуме и панорамировании
- **World space** — мировые координаты, которые видит пользователь в панели свойств

Глобальная точка отсчёта — это где находится **(0, 0) мирового пространства** на экране.

```cpp
class Viewport {
public:
    sf::Vector2f worldOrigin;  // где (0,0) мира на экране (в пикселях)
    float zoom = 1.f;

    // Перевод мировых координат в экранные
    sf::Vector2f worldToScreen(sf::Vector2f world) const {
        return worldOrigin + world * zoom;
    }

    // Перевод экранных координат в мировые
    sf::Vector2f screenToWorld(sf::Vector2f screen) const {
        return (screen - worldOrigin) / zoom;
    }
};
```

Все координаты в **панели свойств** показываются в world space. На холсте фигуры рисуются в screen space через `worldToScreen()`.

#### 10.2 Отображение глобального origin на холсте

Точка (0,0) отображается на холсте как маркер — крестик, как в AutoCAD:

```cpp
void drawWorldOrigin(sf::RenderTarget& target, Viewport& vp) {
    sf::Vector2f pos = vp.worldToScreen({0.f, 0.f});
    // горизонтальная линия
    // вертикальная линия
    // подпись "(0, 0)"
}
```

Опционально — отображать оси X и Y через весь холст (toggle `A`).

#### 10.3 Изменение глобальной точки отсчёта

**Вариант 1 — Сдвиг холста (панорамирование):**
```
Зажать Middle Mouse Button + drag  →  двигается весь холст
  viewport.worldOrigin += mouseDelta
  (все фигуры визуально двигаются, их мировые координаты не меняются)
```

**Вариант 2 — Ручная установка origin:**
В панели или через команду задать конкретную точку (0, 0):
```
┌──────────────────────────────────┐
│  МИРОВОЙ ORIGIN                  │
│  X: [_______]  Y: [_______]      │
│  [Сбросить в центр экрана]       │
│  [Поставить origin в курсор]     │
└──────────────────────────────────┘
```

#### 10.4 Координатная сетка привязана к origin

Сетка рисуется относительно worldOrigin — при панорамировании она движется вместе с холстом, а не висит статично.

### Фаза 11 — Зум (Zoom & Pan) (1 день)

**Цель:** плавное приближение/отдаление холста тремя способами, зум всегда к позиции курсора.

#### 11.1 Логика зума

Зум должен работать **относительно позиции курсора** (как в Figma/AutoCAD) — точка под курсором остаётся на месте:

```cpp
void Viewport::zoomAt(sf::Vector2f screenPoint, float factor) {
    // Точка в мировых координатах до зума
    sf::Vector2f worldPoint = screenToWorld(screenPoint);

    zoom *= factor;
    zoom = std::clamp(zoom, 0.05f, 50.f); // ограничение зума

    // Пересчитать origin так, чтобы worldPoint остался под курсором
    worldOrigin = screenPoint - worldPoint * zoom;
}
```

#### 11.2 Способы управления зумом

**Колёсико мыши:**
```cpp
case sf::Event::MouseWheelScrolled:
    float factor = (event.mouseWheelScroll.delta > 0) ? 1.1f : (1.f / 1.1f);
    viewport.zoomAt(mousePos, factor);
```

**Ctrl + / Ctrl −:**
```cpp
case sf::Event::KeyPressed:
    if (ctrl && key == Equal)  viewport.zoomAt(screenCenter, 1.25f);
    if (ctrl && key == Hyphen) viewport.zoomAt(screenCenter, 0.8f);
    if (ctrl && key == Num0)   viewport.resetZoom(); // сброс к 100%
```

**Кнопки в интерфейсе** (в нижнем правом углу холста):
```
[−]  [ 100% ▾ ]  [+]  [⊡ Fit]
```
- `−` / `+` — шаг ±25%
- `100% ▾` — выпадающий список: 25%, 50%, 75%, 100%, 150%, 200%, Fit to screen
- `⊡ Fit` — вписать все фигуры в экран

#### 11.3 Панорамирование (Pan)

```
Middle Mouse Button + drag  →  панорамирование
  viewport.worldOrigin += mouseDelta

Space + Left Mouse drag  →  панорамирование (как в Figma/Photoshop)
  курсор меняется на "руку" (sf::Cursor::Hand)
```

#### 11.4 Статусная строка

Текущий масштаб отображается в статусной строке внизу:
```
Курсор: (124.5, -87.3)    Масштаб: 150%    Фигура: Прямоугольник
```

### Фаза 12 — Масштабирование фигуры (Scale) (1 день)

**Цель:** равномерно или независимо по осям масштабировать фигуру целиком, не трогая отдельные вершины.

#### 12.1 Хранение масштаба в классе Figure

```cpp
class Figure {
    sf::Vector2f scale = {1.f, 1.f}; // scaleX, scaleY
    // ...
};
```

При вычислении абсолютных координат вершин scale применяется после вращения:
```cpp
sf::Vector2f Figure::getAbsoluteVertex(sf::Vector2f relative) const {
    // 1. Применить scale
    relative.x *= scale.x;
    relative.y *= scale.y;
    // 2. Применить вращение
    float rad = rotationAngle * M_PI / 180.f;
    float rx = relative.x * cos(rad) - relative.y * sin(rad);
    float ry = relative.x * sin(rad) + relative.y * cos(rad);
    return anchor + sf::Vector2f(rx, ry);
}
```

Ось масштабирования — **точка привязки** (как и у вращения).

#### 12.2 Масштабирование мышью через handles

При выделении фигуры на углах и серединах сторон bounding box появляются **8 маркеров масштабирования**:

```
◼ ─── ◼ ─── ◼
│             │
◼             ◼
│             │
◼ ─── ◼ ─── ◼
```

Логика drag по маркеру:
```
Угловой маркер (пропорциональный scale):
  delta = mousePos - startMousePos
  factor = (originalSize + delta) / originalSize
  figure.scale *= factor  // одинаково по X и Y

  Зажат Shift → свободный scale по X и Y независимо

Боковой маркер (scale только по одной оси):
  scaleX или scaleY меняется независимо
```

Курсор меняется в зависимости от маркера: `↔`, `↕`, `↗`, `↙`.

#### 12.3 Ввод масштаба в панели свойств

```
┌─────────────────────────────┐
│  МАСШТАБ                    │
│  W: [_____] %               │
│  H: [_____] %               │
│  🔗 Пропорционально         │  ← замочек, как в Photoshop
└─────────────────────────────┘
```

- По умолчанию замочек **включён** — при изменении W автоматически меняется H пропорционально
- Выключить замочек → можно задать W и H независимо
- 100% = исходный размер фигуры при создании

#### 12.4 Горячие клавиши

| Клавиша | Действие |
|---|---|
| Drag углового маркера | Пропорциональный scale |
| Shift + drag углового маркера | Свободный scale по X/Y |
| Drag бокового маркера | Scale по одной оси |

### Фаза 13 — Сдвиг холста (Pan) и перемещаемая точка отсчёта (1–2 дня)

**Цель:** свободное перемещение по холсту зажатием ЛКМ на пустом месте + возможность зафиксировать точку отсчёта (0,0) в любом месте холста.

#### 13.1 Pan — сдвиг холста через Space + ЛКМ

Зажать `Space` → курсор меняется на руку, теперь ЛКМ + drag двигает холст:

```
KeyPressed (Space):
  курсор → sf::Cursor::Hand

Space зажат + MouseButtonPressed (Left):
  state = PANNING
  panStartPos = mousePos

Space зажат + MouseMoved (while PANNING):
  delta = mousePos - panStartPos
  viewport.worldOrigin += delta
  panStartPos = mousePos

MouseButtonReleased или KeyReleased (Space):
  state = IDLE
  курсор → обычный
```

Это стандартное поведение как в Photoshop и Figma.

#### 13.2 Перемещаемая точка отсчёта (Movable World Origin)

Точка отсчёта — это маркер **(0, 0)** на холсте, который можно **перетащить в любое место**. После этого все координаты всех фигур пересчитываются относительно новой точки. Холст при этом остаётся на месте — двигается только система координат.

**Визуально:** маркер отображается как крестик с кружком, как в AutoCAD:

```
    │
────┼────   ← маркер (0,0), перетаскиваемый
    │
```

**Перемещение маркера мышью:**
```
Hover над маркером (0,0) → курсор меняется на руку

MouseButtonPressed на маркере:
  state = DRAGGING_ORIGIN

MouseMoved (while DRAGGING_ORIGIN):
  delta = mousePos - oldOriginScreenPos  (в мировых координатах)
  worldOriginPoint += screenToWorld(delta)

  // Пересчитать координаты всех фигур:
  for each figure:
    figure.anchor -= worldDelta
    // относительные координаты вершин не меняются

MouseButtonReleased:
  state = IDLE
```

Ключевой момент: фигуры **визуально не двигаются** — двигается только точка (0,0). Координаты в панели свойств обновляются в реальном времени.

**Установка через toolbar** — отдельный инструмент в панели инструментов:

| Иконка | Инструмент | Горячая клавиша |
|---|---|---|
| ⊕ | Переместить точку отсчёта | `O` |

Нажал кнопку в toolbar → курсор сменился на прицел → кликнул на холсте → (0,0) зафиксирован в этом месте → инструмент автоматически переключается обратно на предыдущий.

#### 13.3 Визуальные оси координат

При перемещении origin можно включить отображение осей (toggle `A`):

```
  Y↑
   │
   │
───┼──────────────→ X
   │(0,0)
```

Оси рисуются от origin через весь видимый холст, помогают ориентироваться в координатном пространстве.
